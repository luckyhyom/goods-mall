goods-mall API / errors / FORBIDDEN

FORBIDDEN

HTTP 403 · Forbidden

권한이 없습니다.

의미

인증은 되었지만 해당 리소스에 접근할 권한이 없습니다. 예: USER 토큰으로 관리자 API 호출. RFC 9457 application/problem+json 형식으로 전달됩니다.

발생 조건

  • 관리자(👑) 전용 엔드포인트

응답 예시

HTTP/1.1 403 Forbidden
Content-Type: application/problem+json
{
  "type": "https://docs.goods-mall.dev/errors/FORBIDDEN",
  "title": "Forbidden",
  "status": 403,
  "code": "FORBIDDEN",
  "detail": "권한이 없습니다.",
  "instance": "/api/v1/admin/products"
}

필드

필드의미
type이 페이지의 URI — 에러 종류의 안정적 식별자이자 문서 링크.
title종류별 고정 요약(영어·불변, 로그·관측 그룹핑용).
statusHTTP 상태 코드(403).
code클라이언트가 분기에 사용하는 안정 키. 메시지 대신 이 값으로 처리.
detail이번 발생 건의 사람용 메시지(한국어·가변).
instance에러가 난 요청 경로.

클라이언트 처리

detail을 파싱하지 말고 code로 분기하세요.

if (problem.code === 'FORBIDDEN') {
  // Common 에러 처리
}
